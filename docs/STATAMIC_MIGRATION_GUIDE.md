## Migration Instructions for Ralph

### General Workflow

1. For each post listed above:
   - Create a new entry in Statamic CMS
   - Set the publication date to the **Date** specified in metadata
   - Set the title to the **Title** specified in metadata
   - Set the category to the **Categories** specified (all are "misc" currently)
   - Set the layout to "post"

2. Image Migration:
   - Upload all images listed under **Associated Images** for each post
   - The first image in each list is the **feature image** - set it as the featured/cover image in Statamic
   - Update image references in the post content to match Statamic's asset structure
   - All images are currently referenced using `{{site.url}}/assets/images/posts/...` pattern

3. Content Processing:
   - Read the markdown file from the `_posts` directory
   - Process the frontmatter metadata
   - Convert the markdown content to Statamic's format
   - Update any internal links to match new URL structure

4. Path Mappings:
   - Current Jekyll path: `_posts/YYYY-MM-DD-Post_Name.md`
   - Current image path: `assets/images/posts/[post-folder]/[image-file]`
   - Map these to Statamic's content and asset structure

### Important Notes

- All posts currently use the "misc" category
- All posts use the "post" layout
- Image paths use either direct `/assets/images/posts/` or organized in subdirectories
- Some posts have commented-out image references that may need review
- Post 7 filename contains "copy" - cleanup may be needed
- Post 11 has a future date (2025-04-14) - verify if this is intentional

### Verification Checklist

After migration, verify:

- [ ] All 11 posts are created in Statamic
- [ ] Each post has the correct publication date
- [ ] All feature images are properly assigned
- [ ] All inline images display correctly
- [ ] No broken image links
- [ ] Metadata (title, category, layout) is correct
- [ ] Post content renders properly in Statamic
